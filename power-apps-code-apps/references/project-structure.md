# Power Apps Code Apps Project Structure

## Starter Template Layout

```
my-app/
├── public/                    # Static assets served as-is
├── src/
│   ├── assets/                # Images, fonts, etc. processed by Vite
│   ├── components/            # Reusable React components
│   │   └── ui/                # shadcn/ui components (Button, Card, etc.)
│   ├── hooks/                 # Custom React hooks
│   ├── lib/                   # Utility functions (e.g., cn() for classnames)
│   ├── pages/                 # Route-level page components
│   ├── providers/             # React context providers, state stores
│   ├── App.tsx                # Root component
│   ├── index.css              # Global styles (Tailwind directives)
│   ├── main.tsx               # Entry point — renders App into DOM
│   └── router.tsx             # React Router configuration
├── generated/                 # Auto-generated by pac code add-data-source
│   ├── models/
│   │   └── {Table}Model.ts    # TypeScript interfaces for each table
│   └── services/
│       └── {Table}Service.ts  # CRUD service for each table
├── components.json            # shadcn/ui configuration
├── eslint.config.js
├── index.html                 # HTML entry point
├── package.json
├── power.config.json          # Generated by pac code init (do not edit manually)
├── tsconfig.json
├── tsconfig.app.json
├── tsconfig.node.json
└── vite.config.ts
```

## Technology Stack (Starter Template)

| Layer          | Technology                                        |
| -------------- | ------------------------------------------------- |
| Build          | Vite + `@microsoft/power-apps-vite` plugin        |
| Framework      | React 19                                          |
| Language       | TypeScript 5.9                                    |
| Styling        | Tailwind CSS 4 + shadcn/ui (Radix UI primitives)  |
| Routing        | React Router                                      |
| State          | Zustand                                           |
| Data Fetching  | TanStack Query (React Query)                      |
| Tables         | TanStack Table                                    |
| Icons          | Lucide React                                      |
| SDK            | `@microsoft/power-apps` v1.0.3                    |

## vite.config.ts

```typescript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import tailwindcss from "@tailwindcss/vite";
import powerApps from "@microsoft/power-apps-vite";
import path from "path";

export default defineConfig({
  plugins: [react(), tailwindcss(), powerApps()],
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "./src"),
    },
  },
});
```

The `@` alias maps to `./src`, enabling imports like `import { Button } from "@/components/ui/button"`.

## Key Files

- **power.config.json** — Generated metadata for CLI and SDK. Do not edit or commit secrets here.
- **generated/** — Auto-generated typed models and services from `pac code add-data-source`. Do not hand-edit; re-run the CLI command instead.
- **components.json** — shadcn/ui component registry config. Used by `npx shadcn@latest add <component>`.

## Limitations

- Apps are hosted on publicly accessible endpoints — never store secrets in code
- No Power Platform Git integration
- No Power Apps mobile or Windows app support
- No environment variable support for storing secrets
- Schema changes require deleting and re-adding the data source
